Filing Details

T619 — Transmitter Information

The entity submitting this file to CRA.

Must be 15 chars ending in a program code (e.g. MM0001, RZ0001).
Unique per submission. If you re-submit, you must increment this (e.g. A, B, C).

T4 Summary — Employer Information

The employer who paid the remuneration.

The 15-character Business Number. T4 uses RP; T1204 uses RT; T5018 uses RZ.
Applied to all slips in this submission

Frequently Asked Questions

Is my data safe?

Yes. All processing happens entirely in your browser. Your spreadsheet data is never uploaded to any server. You can verify this by disconnecting from the internet — the tool works offline.

What return types are supported?

We support T4 (Statement of Remuneration Paid), T1204 (Government Service Contract Payments), and T5018 (Statement of Contract Payments). NR4 and other returns are coming soon.

What format should my Excel file be in?

For T4: Your spreadsheet needs columns for: Last Name, First Name, SIN, Employment Income, CPP Contributions, EI Premiums, Income Tax Deducted, EI Insurable Earnings, CPP/QPP Pensionable Earnings, and Province of Employment. Optional: Union Dues, RPP Contributions, CPP2, QPP, Address.
For T1204: Vendor Name, Vendor BN or SIN, Amount Paid, Business Type, Service Payments Only (Yes/No), and Address. The tool auto-detects common header names.

What CRA spec version does this support?

The generated XML conforms to the T619 2026 / T4 2026V4 / T1204 2026V3 / T5018 2026V3 specification, including the Oct 2025 validation change requiring empty optional tags to be removed.

How do I submit the XML to CRA?

Upload the downloaded XML file through CRA's Internet File Transfer (IFT) application. You'll need a CRA My Business Account or Represent a Client account.

Why did you build this?

As an automation consultant at AVANZA Business Automation, I constantly saw small businesses and accountants struggling with CRA's strict XML filing requirements. If you file more than 5 information slips, CRA forces you to submit them electronically in an XML format. Because XML is notoriously difficult for non-programmers to generate from Excel, businesses are often forced to buy bloated, expensive payroll software just for this one feature. I built TaxXML.ca as a fast, free, and secure alternative to democratize tax filing for Canadian businesses.
